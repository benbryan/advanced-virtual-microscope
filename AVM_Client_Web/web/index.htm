Overview

Advanced Virtual Microscope (AVM) is an image annotation and analysis platform for working with gigapixel images.  AVM allows users to generate annotations on very large images which are uploaded and stored on a remote server for collaboration or authentication. Feature generators are provided and can be used to train and test classifiers provided by Weka. The generated feature values can be displayed for testing how different features distinguish various image sections. Custom plug-ins for feature generation or classification are supported for community development.

Features are generated within AVM from a sliding window. Tiles (sub images) of variable size D are sampled within selected regions. Each tile can be padded with bordering pixels  up to size W to generate cleaner features. 

note: right click context menus are used throughout AVM for additional options

Starting AVM

AVM uses Java Network Launch Protocol (JNLP) files to download and execute AVM on your local machine. Some users will be able to click on the Java "Launch" button (seen above) and have AVM start after passing a few security prompts. Other users will see a Java splash screen appear then disappear with no information as to what happened.  This issue could be caused by one of many things listed below

1.       You tried to execute the 64-Bit version through a 32-Bit browser.
Unfortunately most browsers are still 32-Bit and therefore cannot use the 64-Bit java runtime. Since the 64-Bit version of AVM requests a heap size larger than the 32-Bit runtime can support, the execution silently crashes.  Try the 32-Bit version.
To use the 64-Bit version you may need to execute the JNLP file through the command line using javaws.exe. First download the JNLP file to your computer then type "javaws AVM_X64.jnlp" in a command prompt in same folder.   

2.       You have a cached version of AVM causing errors.
Try clicking on the same launch button more than once. The JNLP file is set to update AVM on execution. Unfortunately the update is not applied until the next execution.
Try clearing your Java cache

3.       Sometimes you must fully reboot your computer for Java to work.

Client Application

The AVM client application views and manages images, studies, and solutions. All of the images available on the server can be opened and viewed by clicking on File->Image->Open.  This will open a seperate window where you can pan and zoom around the image, but nothing else. Other options in this sub menu include Upload and Delete. Upload includes a how to description within it, and Delete will only allow you to delete images you uploaded.  The final menu item "Open Local" is good for testing if AVM is compatible with images on your computer.

Study Manager

Studies are collections of images with annotated regions of interest (ROIs). Studies can be accessed from AVM's main window under the File->Study menu. To create a study click on File->Study->New. After creating the study you will be prompted to select images for the study.

The clone button on the bottom of the Study Manager creates a copy of the study with a new name.  This function can be used for creating a backup or branch of the current study.

After a study has been created and populated with images you can open the images by double clicking on the image name. Images will be opened in a new window (described below) for annotation.

 

Image Viewer

An image viewer by default has only a thumbnail. Within the thumbnail users can view the full image and quickly pan around using the left mouse button. The rest of the window displays a subsection of a the image which can be zoomed and panned around in using a mouse and scroll wheel, or the "+/-" keyboard keys and arrows. When working on studies or solutions the window will also contain an ROI manager for use with the toolbar in the main AVM window.  Various left mouse button functions listed below . Regardless of which tool is selected, the right mouse button can always be used for click and drag panning within the image.

Users generated ROIs are grouped into folders which can be renamed, deleted, etc using context menus accessed by right clicking on the folder name.

Image Pan

When selected, the left mouse button can be used to pan around the image by click and drag.

Select ROIs

Clicking inside an ROI will highlight it with red markers in the main window. This will also highlight the ROI in the ROI and Solution Managers. Click multiple times on the same spot to cycle through overlapping ROIs.

Select Point

ROIs can be manipulated using this tool by click and dragging individual points. Only operates on ROIs selected in the ROI Manager

Freehand ROI

Freehand ROIs by click and drag. Users can create very large and detailed ROIs by using both the left and right mouse buttons to annotate and pan at the same time.

Oval ROI

Create simple oval shaped ROIs

Rectangular ROI

Create variable or fixed size rectangular ROIs. Right clicking on this tool's icon will present a drop down menu with the option "Setup." Selecting this item will display a dialog where a fixed size restraint can be specified and enabled or disabled.

Intelligent Scissors (IS)

Based on "Intelligent Scissors for Image Composition" by Eric Mortensen and William Barrett

This tool uses creates a "Live-Wire" that attempts to lock onto boundaries within an image. The processes begins by supplying a seed node on an image by left clicking. Between this point and the cursor a live wire is formed. As the cursor moves the live wire is updated. Once the live wire snaps to a desired path, a second left click locks it in place and generates a new seed node. The segment can be finalized by double left clicking or clicking back on the first seed node. In case the live wire does not snap to the desired boundary, a click and drag will create a freehand segment. Also, during segmentation a right click and drag can be used to pan around the image.

Settings Panel

To open the settings panel, right click on the IS icon.

Feature Weights are as defined in "Intelligent Scissors for Image Composition." Feel free to adjust them randomly to improve segmentation performance on your images.

·         w_z is the Laplacian of Gaussian (LOG) zero crossing feature weight

·         w_g is the Gradient magnitude feature weight

·         w_d is the Gradient direction feature weight

Checkbox options

·         Work on base resolution forces IS to operate on the highest resolution images available. When unchecked IS operates on images used displayed on the users monitor. When checked IS may need to fetch the higher resolution images, and may function very slowly.

·         Approximate inverse cosine
The gradient direction feature requires an inverse cosine in its calculations. When checked, IS uses a fast polynomial approximation rather than Java's very slow acos function.

Solution Manager

The solution manager is an extension of the study manager.  Effectively a solution contains a study, a feature generator, and a classifier. The study in the solution has a set of classes instead of ROI folders. the folder names are common throughout all images within the same study. Creating a new class or deleting an old propagates through all images within the solution. 

A temporary folder is available for generating ROIs that will not be saved. Any ROIs within this folder are discarded when the image is closed.

To get started using the solution manager follow the steps below

1.       First create a new solution by clicking on File->Solution->New

2.       Select which features and classifier you wish to use, or click "Accept." You can always change these parameters later by clicking Classifier->"Setup Model."

3.       Select images to add to the solution.

4.       Open the images and generate ROIs at least two classes

5.       Click Classifier->"Generate Model" to generate features and train the selected classifier

6.       Create and select a new region in the Temporary folder to test the classifier model.

7.       Press the "Classify" button to generate an overlay showing the class labels.

Implementation

The AVM system has 3 major components; Java web services, a database, and a client application.  The web services are designed to run on Glassfish servers and provide CRUD interfaces between client applications and a NoSQL database.  The server can also perform computationally intensive operations for the client application. It also hosts the client application for download and execution on the clients machine.  The client applications primary function is to allow users to rapidly interact with very large images.  The database manages these images in a Flashpix style format where large images are stored in small individual compressed tiles at several different resolutions. When clients are viewing images only the small sub regions required for display on a users monitor are downloaded.  Image annotations and other data are also stored in the database and are accessed as needed by the client.

NoSQL Database

NoSQL databases store information as key-value pairs.  Keys consist of major and minor components the combination of which dictates how data is clustered in the database.  All keys with the same major key are stored close to each other for fast access across the major key set.  The values assigned to these keys are byte arrays which can be assigned Apache Avro schemas if desired.

AVM's database layout follows the schema seen below. The most complicated part of the database involves the layout of images.  When images are imported into the database, the image is first converted into a layered and tiled JPEG-in-TIFF image.  Each layer is a sub-sampled version of the original with samplings ratios following sequence {1/1, 1/4, 1/16...}.  Tiles are 256 by 256 pixels sub regions of each layer and are stored in the same format as they exist in the TIFF image.  Each tile in each layer is accessed with a unique key and can therefore be independently retrieved.  The keys major components include the images name and group while minor components include the layer (sub-sampled image index) and XY coordinates of the tile within the layer. Tile coordinates are indexed as {0,1,2} and not as multiples of the tiles size.  Each layer is stored with a set of properties similar to TIFF tags, and each property is stored as a separate database entry.

When a client generates specific features for a specific image the client first checks the database for preexisting feature data. If the requested features already exist then they are pulled from the database and sent to the client. Otherwise the client must download the image data and generate the features on their machine.  At that same time, the features requested are queued to be generated across the whole image then stored in the database for later use.
